JWT(JSON Web Token)
데이터가 json으로 이루어져 있는 토큰
두 개체가 서로 안전하게 정보를 주고받을 수 있도록 웹 표준으로 정의된 기술

토큰 기반 인증 시스템

토큰:
- 로그인 이후 서버가 만들어 주는 문자열
- 문자열 : 사용자의 로그인 정보(서버에서 발급 되었음을 증명하는 서명도 포함)

사용자가 로그인을 하면 서버에서 해당 사용자의 정보를 지니고 있는 토큰을 발급
추후 사용자가 다른 api를 요청하게 될 때 발급받은 토큰과 함께 요청
서버는 해당 토큰이 유효한지 검사
결과에 따라 작업을 처리하고 응답

회원가입과 로그인에 성공했을 때 토큰을 사용자에게 전달해줘야된다.
사용자가 브라우저에서 토큰을 사용할 때 주로 두 가지 방법을 사용
	1) 브라우저의 localStorage or sessionStorage에 담아서 사용하는 방법
		but.. XSS 위험(악성 스크립트 삽입)
	2) 쿠키에 담아서 사용하는 방법
		XSS위험이 있지만 httpOnly라는 속성을 활성화하면 자바스크립트를 통해 		쿠키를 조회할 수 없으므로 악성스크립트로부터 안전
		but.. CSRF(cross site request forgery) - 토큰을 쿠키에 담으면 사용자가 		서버로 요청을 할 때마다 무조건 토큰이 함께 전달되는 점을 이용해서 사		용자가 모르게 원하지 않는 API 요청을 하게 만든다
			-> CSRF 토큰 사용 및 Referer 검증 등의 방식으로 막을 수 있다.

jwtMiddleware
iat: 토큰이 언제 만들어졌는지
exp: 언제 만료되는지
